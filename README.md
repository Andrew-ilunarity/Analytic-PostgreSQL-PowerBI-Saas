📊 CASE: Наскрізна SaaS-аналітика відтоку клієнтів (250K+ транзакцій)
📌 S (Situation) / Ситуація
Продуктова SaaS-компанія з базою у 250 000+ клієнтів зіткнулася з проблемою неконтрольованого відтоку користувачів (Churn) та падінням щомісячного прогнозованого доходу (MRR). Початковий датасет, вивантажений із серверів, містив величезну кількість системного «сміття», аномалій та пропусків даних (Data Pollution), що унеможливлювало побудову точних звітів у BI-системах. Керівництву компанії був необхідний прозорий інструмент для пошуку фінансових дірок, аналізу тривалості життя клієнтів (Retention) та виявлення поведінкових факторів ризику.

🎯 T (Task) / Завдання
Як єдиний дата-аналітик на проекті, я мав розробити наскрізну аналітичну систему «від сирих даних до бізнес-інсайтів».
Основні виклики:

Симуляція та очищення даних: Створити Python-скрипт для генерації аномалій (щоб протестувати стрес-стійкість пайплайну), а потім засобами SQL повністю очистити базу.

Оптимізація архітектури (Heavy ETL / Light BI): Уникнути важких обчислень на стороні Power BI (DAX-міри на 250К рядків уповільнюють роботу) і перенести всю математику групування на сторону сервера PostgreSQL.

Побудова веб-інтерфейсу: Розробити дворівневий інтерактивний дашборд для СЕО та продуктової команди.

🛠️ A (Action) / Дії
1. Етап Data Engineering (Python & PostgreSQL):
Написав скрипт на Python (Pandas, Numpy) для контрольованого внесення «бруду» в дані: згенерував близько 22 000 пропусків (NULL) у фінансах та статі, зашив аномальний вік (0, -5, 150 років) та порушив текстовий регістр контрактів із додаванням прихованих пробілів на кінці.

У PostgreSQL написав складний скрипт очищення даних, загорнутий у CREATE VIEW. Використав аналітичні конструкції CTE (Common Table Expressions) для ізольованого розрахунку «чистого середнього віку» без урахування аномалій та підставив його через CASE WHEN.

За допомогою текстових функцій (LOWER(), LIKE '%...%') нормалізував хаотичні категорії контрактів, звівши їх до єдиного стандарту.

2. Етап Оптимізації архітектури (Data Marts):
Замість завантаження 250 000 сирих рядків у BI, я спроектував агреговані аналітичні вітрини даних на стороні сервера (GROUP BY за перетином ключових зрізів: місяці, стать, регіон, тип інтернету). Це зменшило обсяг передаваних даних у рази, дозволивши дашборду синхронізуватися за мілісекунди.

Написав аналітичні запити для когортного аналізу (динаміка життя клієнта від 1 до 72 місяців) та умовної агрегації фінансових показників.

3. Етап Візуалізації та UX/UI дизайну (Power BI):
Розробив інтерактивний інтерфейс у стилі сучасного веб-додатку (мінімалістична світла тема, модульна сітка без стандартних громіздких рамок). Дашборд було розбито на дві логічні сторінки:

Сторінка 1 (Когортний аналіз та MRR): Побудував лінійні графіки утримання користувачів (Retention/Churn Cohorts), інтегрував динамічні випадні зрізи, які миттєво перераховували фінансовий стан компанії.

Сторінка 2 (Risk Metrics & Поведінка): Створив діаграму розсіювання (Scatter Chart) для виявлення кластерів споживання трафіку, стовпчасті діаграми для аналізу скарг та умовне колірне форматування (Heatmap) для матриці ризиків.

🏆 R (Result) / Результат
Створено надійний ETL-пайплайн: База даних об'ємом 250K рядків повністю очищена від аномалій та приведена до єдиного аналітичного стандарту.

Висока продуктивність дашборду: Завдяки переносу логіки агрегації в PostgreSQL, дашборд працює в режимі реального часу без зависань, а важкі DAX-обчислення зведені до мінімуму.

Виявлено критичні бізнес-інсайти для СЕО:

Проблема першого року: Аналіз когорт чітко показав, що в перші 12 місяців компанія втрачає 50% клієнтів (проблема онбордингу), але після 12-го місяця відтік різко падає і стабілізується на рівні 26-27% (вища лояльність).

Технічний ризик: Графік категорій мережі виявив, що найбільший обсяг фінансових втрат та відтоку генерують користувачі технології Fiber Optic (сигнал для перевірки технічного відділу).

Операційний ризик: Підсвічено понад 16 000 випадків прострочення платежів, що дозволило компанії скоригувати роботу системи автоматичних нагадувань.
